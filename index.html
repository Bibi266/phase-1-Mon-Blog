Comprendre le chaînage optionnel en JavaScript

Aujourd’hui, j’ai appris une astuce super pratique en JavaScript : le chaînage optionnel. Il permet d’accéder aux propriétés d’un objet sans risquer une erreur si une partie est undefined. Exemple : 

const user = { profile: { name: "Roberte" } };
console.log(user?.profile?.name); // "Roberte"
console.log(user?.contact?.email); // undefined, mais pas d’erreur !

C’est très utile quand on manipule des objets complexes ou des données venant d’une API.
